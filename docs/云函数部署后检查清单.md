## 一、每个云函数部署时要确认什么

对每一个云函数，都按同样标准检查：

1. 在微信开发者工具里右键云函数目录
2. 选择“上传并部署：云端安装依赖”
3. 等待部署完成，不要中途关闭窗口
4. 查看结果提示是否明确显示成功
5. 如果有“查看日志”或“查看输出”，确认没有 `Cannot find module 'wx-server-sdk'` 之类报错

只要出现下面任意一种情况，就不能算部署成功：

- 只上传了，但没有部署
- 部署中断
- 提示依赖安装失败
- 提示找不到 `wx-server-sdk`
- 提示云环境无权限或环境不存在

## 二、推荐部署顺序

按这个顺序来，排错最省时间：

1. `getDemoData`
2. `login`
3. `createNotifyTask`
4. `sendNotify`

原因：

- `getDemoData` 最容易验证，能最快判断云调用链路是否通
- `login` 会碰 `users` 集合
- `createNotifyTask` 和 `sendNotify` 会碰 `notifications` 集合

## 三、每个函数部署后怎么验证

### 1. `getDemoData`

这是最先验证的函数。

部署后检查：

- [ ] 部署成功提示正常
- [ ] 没有依赖安装报错
- [ ] 首页或项目页顶部状态能从 `Mock Demo` 变成 `CloudBase 已连接`

如果页面状态已经变成 `CloudBase 已连接`，说明至少这几件事是通的：

- `envId` 正确
- `useMock` 已关闭
- 云函数调用链路可用

### 2. `login`

部署后检查：

- [ ] 部署成功
- [ ] `users` 集合已经创建
- [ ] 后续如果调用该函数，不会报集合不存在

当前阶段不要求页面立刻调用它，但它必须能正常部署。

### 3. `createNotifyTask`

部署后检查：

- [ ] 部署成功
- [ ] `notifications` 集合已经创建
- [ ] 没有缺少依赖或数据库权限报错

### 4. `sendNotify`

部署后检查：

- [ ] 部署成功
- [ ] `notifications` 集合已经创建
- [ ] 没有数据库查询报错

## 四、最小成功标准

在当前阶段，你不需要四个函数都跑业务，只需要满足下面这组最小标准：

- [ ] 四个云函数都部署成功
- [ ] 首页或项目页顶部显示 `CloudBase 已连接`
- [ ] 控制台没有 `CloudBase unavailable`
- [ ] 控制台没有 `Cannot find module 'wx-server-sdk'`

满足这 4 条，就可以认为“云环境接通”这一步完成。

## 五、常见错误与对应处理

### 报错：`Cannot find module 'wx-server-sdk'`

说明：
云函数依赖没有装上。

处理：

1. 不要用普通上传
2. 改用“上传并部署：云端安装依赖”
3. 重新部署该函数

### 报错：环境不存在 / 环境未初始化

说明：
当前开发者工具选中的云环境，和 [cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js) 里的 `envId` 不一致，或者 `envId` 填错了。

处理：

1. 检查开发者工具当前绑定环境
2. 检查 [cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js) 的 `envId`
3. 保存后重新编译
4. 重新部署函数

### 报错：集合不存在

说明：
对应数据库集合还没创建。

处理：

1. 打开云数据库
2. 创建对应集合
3. 再重新测试函数

对应关系：

- `login` 需要 `users`
- `createNotifyTask` 需要 `notifications`
- `sendNotify` 需要 `notifications`

### 页面还能打开，但状态一直是 `Mock Demo`

说明：
项目还在走本地回退逻辑。

优先检查：

1. [cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js) 里的 `useMock` 是否是 `false`
2. `envId` 是否已经替换
3. `getDemoData` 是否部署成功

## 六、你现在就按这个顺序做

1. 先部署 `getDemoData`
2. 打开首页，看顶部状态
3. 如果已显示 `CloudBase 已连接`，继续部署其余 3 个函数
4. 如果还不是 `CloudBase 已连接`，先不要继续，先排查 `envId`、`useMock`、当前云环境
