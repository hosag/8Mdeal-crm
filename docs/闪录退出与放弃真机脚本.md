# 闪录退出与放弃真机脚本

这是一份现场点测脚本。

使用方式：

1. 拿真机打开首页
2. 按顺序执行下面每一步
3. 每步只看“结果应该是什么”
4. 任何一步不符合，就记录成 bug

## 测试 1：空白态右上角关闭

动作：

1. 打开首页闪录
2. 什么都不做
3. 点右上角 `关闭`

应该看到：

- 直接关闭
- 不出现三选一
- 不出现 `已暂存，下次可继续`

## 测试 2：空白态点遮罩

动作：

1. 再打开首页闪录
2. 什么都不做
3. 点遮罩

应该看到：

- 直接关闭
- 不出现 `已暂存，下次可继续`

## 测试 3：手动输入后点取消

动作：

1. 打开闪录
2. 点击 `不方便说？改为手动输入`
3. 输入：`客户说周五看方案`
4. 点底部 `取消`

应该看到：

- 出现三选一
- 有 `继续编辑`
- 有 `保存草稿，稍后继续`
- 有 `放弃本次闪录`

## 测试 4：保存草稿后恢复

动作：

1. 接上一步
2. 选择 `保存草稿，稍后继续`
3. 再次打开闪录

应该看到：

- 关闭时提示 `已暂存，下次可继续`
- 刚才输入的 `客户说周五看方案` 还在

## 测试 5：放弃草稿后彻底清空

动作：

1. 保持当前有内容
2. 点 `取消`
3. 选 `放弃本次闪录`
4. 在确认弹层中再点确认
5. 重新打开闪录

应该看到：

- 有二次确认
- 提示 `已放弃本次闪录`
- 重新打开后内容为空
- 不会恢复上一条草稿

## 测试 6：遮罩关闭有内容草稿

动作：

1. 打开闪录
2. 手动输入：`周三回访预算`
3. 不点取消，直接点遮罩关闭
4. 再次打开闪录

应该看到：

- 直接关闭
- 提示 `已暂存，下次可继续`
- 再打开时 `周三回访预算` 还在

## 测试 7：录音中点取消

动作：

1. 打开闪录
2. 开始录音
3. 在录音还没结束时点 `取消`

应该看到：

- 不会直接退出
- 会出现三选一
- 第二个选项优先应是 `直接关闭`，而不是假装 `保存草稿，稍后继续`

继续动作：

1. 再来一次
2. 录音中点 `取消`
3. 选 `放弃本次闪录`

应该看到：

- 录音被停止
- 本次状态被清空
- 不残留录音波形

## 测试 8：识别中点取消

动作：

1. 录一条短语音
2. 等待页面进入 `识别中`
3. 点 `取消`

应该看到：

- 出现三选一
- 不应误导为“肯定已暂存”

继续动作：

1. 选 `放弃本次闪录`

应该看到：

- 当前识别流程停止
- 重新打开时，这条内容不残留

## 测试 8B：识别中点遮罩

动作：

1. 再录一条短语音
2. 等待页面进入 `识别中`
3. 不点 `取消`，直接点遮罩

应该看到：

- 不会静默直接消失
- 如果当前还不能形成可恢复草稿，应出现退出承接
- 不应误提示 `已暂存，下次可继续`

## 测试 9：理解中点取消

动作：

1. 再录一条语音
2. 等待页面进入 `理解中`
3. 点 `取消`

应该看到：

- 出现三选一
- 不会直接丢失且无提示

继续动作：

1. 选 `放弃本次闪录`

应该看到：

- 当前理解流程停止
- 不残留 `理解中` 状态

## 测试 10：识别结果已出但未保存

动作：

1. 录一条能正常识别的语音
2. 等识别结果出现
3. 不保存，点右上角 `关闭`

应该看到：

- 出现三选一

继续动作 A：

1. 选 `保存草稿，稍后继续`
2. 再打开闪录

应该看到：

- 识别后的文本还在

继续动作 B：

1. 再次点 `关闭`
2. 选 `放弃本次闪录`
3. 再打开闪录

应该看到：

- 刚才识别出来的文本不在了

## 测试 11：已有继续关联时直接放弃

动作：

1. 让闪录回到极简初始态
2. 确保页面上出现 `将继续关联`
3. 点击 `不继续这次关联，清空本次闪录`

应该看到：

- 出现放弃确认

继续动作：

1. 确认放弃
2. 重新打开闪录

应该看到：

- 刚才那个继续关联项目已被清空
- 不再默认沿用

## 测试 12：已选项目但未提交

动作：

1. 输入：`客户周五看方案，下周回我`
2. 手动选择一个项目
3. 点 `取消`

应该看到：

- 出现三选一

继续动作 A：

1. 选 `保存草稿，稍后继续`
2. 再打开闪录

应该看到：

- 输入内容还在
- 已选项目还在

继续动作 B：

1. 点 `取消`
2. 选 `放弃本次闪录`
3. 再打开闪录

应该看到：

- 输入内容没了
- 已选项目也没了

## 测试 13：保存成功后的关闭

动作：

1. 正常保存一条闪录
2. 进入成功面板
3. 点击 `关闭`

应该看到：

- 正常关闭成功面板
- 不出现三选一

## 最后结论

如果下面 5 条都满足，这轮退出承接就基本可以提测：

- `关闭`、`暂存`、`放弃` 三件事用户看得懂
- `取消` 不再偷偷等于“存草稿”
- 点遮罩不会误丢内容
- `放弃本次闪录` 后真的不会再恢复回来
- `将继续关联` 状态下用户也能明确退出
