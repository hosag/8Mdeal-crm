# 闪录退出与放弃测试清单

这份清单只针对首页 `闪录` 的退出承接逻辑。

目标很明确：

- 该暂存时，系统要明确暂存
- 该关闭时，系统不要假装暂存
- 该彻底放弃时，用户必须能真正清空

## 一、这轮重点验什么

- [ ] `遮罩点击` 和 `取消 / 关闭` 是否已经不是同一路径
- [ ] 有内容时，`取消 / 关闭` 是否会出现三选一承接
- [ ] 选择 `保存草稿，稍后继续` 后，下次打开是否能恢复
- [ ] 选择 `放弃本次闪录` 后，这次内容是否真的被清空
- [ ] 只有录音中、识别中但还没形成可恢复草稿时，是否不会误提示“已暂存”
- [ ] 极简初始态下如果显示“将继续关联”，是否也能直接清空本次关联

## 二、测试前准备

建议准备两类数据：

1. 至少 1 个已有项目，用于验证“将继续关联”
2. 至少 1 条真实闪录语音，用于验证“识别后未保存”

建议先确保：

- [ ] 真机录音可用
- [ ] 闪录项目匹配链路可正常工作
- [ ] 首页闪录已有至少 1 个最近项目可供继续关联

## 三、核心场景

### 场景 A：空白态直接关闭

操作：

1. 打开首页闪录
2. 不说话，不输入，不选项目
3. 点右上角 `关闭`

预期：

- [ ] 直接关闭
- [ ] 不出现三选一
- [ ] 不出现“已暂存，下次可继续”
- [ ] 下次再打开时仍是全新空白态

### 场景 B：空白态点击遮罩

操作：

1. 打开首页闪录
2. 不说话，不输入，不选项目
3. 点击遮罩关闭

预期：

- [ ] 直接关闭
- [ ] 不出现暂存提示
- [ ] 下次打开仍为空白态

### 场景 C：只输入一点手动内容，再点取消

操作：

1. 打开闪录
2. 手动输入一句跟进内容，例如：`客户说周五看方案`
3. 点击底部 `取消`

预期：

- [ ] 出现三选一承接
- [ ] 选项包含 `继续编辑`
- [ ] 选项包含 `保存草稿，稍后继续`
- [ ] 选项包含 `放弃本次闪录`

### 场景 D：选择“保存草稿，稍后继续”

操作：

1. 接场景 C
2. 在三选一里点 `保存草稿，稍后继续`
3. 重新打开闪录

预期：

- [ ] 关闭时出现 `已暂存，下次可继续`
- [ ] 再次打开后，刚才输入的内容仍在
- [ ] 未完成的项目关联状态仍在

### 场景 E：选择“放弃本次闪录”

操作：

1. 输入内容或完成一次语音识别
2. 点击 `取消`
3. 在三选一里点 `放弃本次闪录`
4. 在确认弹层中再次确认
5. 重新打开闪录

预期：

- [ ] 会出现二次确认，不会一键误删
- [ ] 关闭后提示 `已放弃本次闪录`
- [ ] 再次打开时，内容已被清空
- [ ] 语音识别结果已被清空
- [ ] 已选项目已被清空
- [ ] 不会再恢复刚才那条草稿

### 场景 F：点击遮罩关闭有内容的草稿

操作：

1. 输入一句内容，例如：`周三回访预算`
2. 不点取消，直接点击遮罩
3. 再次打开闪录

预期：

- [ ] 直接关闭，不出现三选一
- [ ] 出现 `已暂存，下次可继续`
- [ ] 再次打开后，内容仍在

### 场景 G：录音中点取消

操作：

1. 开始语音闪录
2. 在录音还没结束前点 `取消`

预期：

- [ ] 不会直接关闭并丢失
- [ ] 会出现三选一承接
- [ ] 若当前还没有形成可恢复草稿，第二个选项应显示为 `直接关闭`，而不是 `保存草稿，稍后继续`
- [ ] 若选择 `放弃本次闪录`，应停止录音并清空本次状态

### 场景 H：识别中 / 理解中点取消

操作：

1. 录音结束后，等待进入 `识别中` 或 `理解中`
2. 点击 `取消`

预期：

- [ ] 会出现三选一承接
- [ ] 不应错误提示“草稿一定已保存”
- [ ] 选择 `放弃本次闪录` 后，应停止当前处理并清空本次状态

### 场景 H2：识别中 / 理解中点遮罩

操作：

1. 录音结束后，等待进入 `识别中` 或 `理解中`
2. 不点 `取消`，直接点遮罩

预期：

- [ ] 不应静默关闭并直接丢失当前状态
- [ ] 如果当前还不能形成可恢复草稿，也应出现退出承接，而不是直接消失
- [ ] 只有在确实可恢复时，才允许按“暂存并关闭”处理

### 场景 I：识别结果已出来，但还没保存

操作：

1. 完成一次语音识别
2. 保持在“已识别、未保存”的状态
3. 点右上角 `关闭`

预期：

- [ ] 会出现三选一承接
- [ ] 选择 `保存草稿，稍后继续` 后，下次可恢复识别后的文本
- [ ] 选择 `放弃本次闪录` 后，识别结果不会残留

### 场景 J：已有“将继续关联”时直接清空

操作：

1. 让闪录处于极简初始态
2. 页面出现 `将继续关联`
3. 点击 `不继续这次关联，清空本次闪录`

预期：

- [ ] 会出现放弃确认
- [ ] 确认后当前继续关联项目被清空
- [ ] 再次打开或回到初始态时，不再沿用刚才的关联项目

### 场景 K：已选项目但未提交

操作：

1. 输入内容
2. 选中一个项目
3. 不保存，直接点击 `取消`

预期：

- [ ] 放弃确认文案里应体现“已关联项目”这个状态
- [ ] 选择 `保存草稿，稍后继续` 后，下次打开仍能看到已选项目
- [ ] 选择 `放弃本次闪录` 后，项目关联必须清空

### 场景 L：保存成功后的关闭不受影响

操作：

1. 正常保存一条闪录
2. 在成功面板里点击 `关闭`

预期：

- [ ] 不出现三选一
- [ ] 正常关闭成功面板
- [ ] 已保存内容不受影响

## 四、失败链路也要测

下面几条至少故意打一次：

- [ ] 点遮罩关闭时，若当前并不能形成草稿，不应误提示 `已暂存`
- [ ] 放弃后重新打开，不能出现“上一条又回来了”
- [ ] 录音中放弃后，不能继续残留录音波形或处理中状态
- [ ] 已有继续关联项目时放弃，不能只清内容不清项目

## 五、通过标准

满足下面这组条件，就可以认为这轮退出承接达到可提测：

- [ ] 用户能明确区分 `关闭`、`暂存`、`放弃`
- [ ] 用户不会因为点了 `取消` 却被偷偷存草稿而困惑
- [ ] 用户不会因为点了遮罩却误丢内容
- [ ] “将继续关联”状态下，用户有明确的放弃出口
- [ ] 放弃动作能真正清掉内容、识别结果、项目关联和草稿

## 六、最值得优先回归的 5 条

每次继续改闪录后，优先回归这 5 条：

1. 空白态点 `关闭` 是否还会误出三选一
2. 输入一句内容后点 `取消`，是否会出现 `继续编辑 / 保存草稿 / 放弃`
3. 录音中点 `取消`，第二个选项是否会根据可恢复性显示为 `直接关闭` 或 `保存草稿`
4. `放弃本次闪录` 后，重新打开是否绝对空白
5. 极简态 `将继续关联` 下，是否真的可以一键放弃当前关联
