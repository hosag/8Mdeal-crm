# 视觉主题与阅读模式实施清单

本文档用于定义成交 CRM 小程序的“视觉主题”与“字体 / 阅读模式”能力，目标不是简单增加几套背景色，而是建立一套可持续扩展的个性化外观系统。

本文档包含：

- 目标与边界
- 主题命名与风格方向
- 字体与阅读模式规则
- 全局 Token 设计
- 数据模型与云函数改造
- 页面改造范围
- 实施顺序与验收标准

适用前提：

- 云环境已接通
- [getUserPreferences](/Users/shaominhe/成交CRM-CodeX版/cloudfunctions/getUserPreferences/index.js) 与 [saveUserPreferences](/Users/shaominhe/成交CRM-CodeX版/cloudfunctions/saveUserPreferences/index.js) 已上线
- 首页、我的项目、项目详情、跟进录入、我的、收益统计 已具备正式使用级别界面

## 一、目标与原则

### 1.1 目标

本能力的目标不是“让界面看起来更花”，而是同时解决 3 个问题：

1. 让用户能按照自己的审美偏好选择界面气质
2. 让不同年龄、不同视力条件的用户都能舒适使用
3. 建立一套后续可持续扩展的主题系统，而不是每改一页都手工重写颜色

### 1.2 核心原则

- `视觉主题` 与 `字体 / 阅读模式` 必须分离，允许自由组合
- 主题切换优先通过全局 Token 实现，不做页面级硬编码换皮
- 阅读模式不只是放大字体，还应同步影响行高、控件高度和点击区
- 第一阶段优先改造核心高频页面，不追求一次覆盖所有页面
- 节日主题作为增强项预留，不阻塞常驻主题上线

### 1.3 不建议的做法

不建议：

- 直接把主题命名成“商务男士 / 商务女士 / 小女生 / 中老年人”
- 仅通过更换背景图实现所谓主题
- 只做“大字体”而不调整行高和控件尺寸
- 每个页面单独维护一套主题分支样式

## 二、能力拆分

本能力应拆成两组独立设置：

### 2.1 视觉主题

负责控制：

- 页面背景氛围
- Hero 区颜色与层次
- 卡片背景与阴影
- 强调色、状态色、胶囊色
- 整体气质与品牌观感

### 2.2 字体与阅读模式

负责控制：

- 页面基础字号
- 标题与正文层级
- 行高
- 按钮高度
- 胶囊高度
- 卡片内边距与可点击区域

## 三、主题命名与方向

### 3.1 第一阶段建议上线的常驻主题

第一阶段建议先做 4 个常驻主题，覆盖不同典型偏好。

#### 主题 A：深海商务

适用心理预期：

- 商务男士
- 偏稳重、克制、专业

视觉方向：

- 深蓝灰
- 冷金属灰
- 米金强调

关键词：

- 稳
- 干净
- 冷静
- 高级

#### 主题 B：米杏雅致

适用心理预期：

- 商务女士
- 偏温和、精致、轻商务

视觉方向：

- 暖米白
- 杏灰
- 烟粉灰点缀

关键词：

- 温润
- 精致
- 柔和
- 不甜腻

#### 主题 C：樱雾轻盈

适用心理预期：

- 年轻女性
- 偏轻松、亲和、轻快

视觉方向：

- 雾粉
- 奶白
- 低饱和豆沙色

关键词：

- 轻盈
- 明快
- 柔软
- 有亲和力

#### 主题 D：墨竹易读

适用心理预期：

- 中老年用户
- 更重识别度与稳定感

视觉方向：

- 高对比墨绿灰
- 米白底
- 深文字色

关键词：

- 易读
- 稳定
- 对比清晰
- 少装饰

### 3.2 节日主题位

第一阶段不做多套节日主题，只预留一个 `节日限定` 主题位。

使用原则：

- 平时默认关闭
- 节日期间可按版本切换
- 不额外增加一套配置模型

建议节日位支持：

- 春节
- 中秋
- 新年

## 四、字体与阅读模式

### 4.1 第一阶段建议上线的模式

#### 模式 A：默认

用途：

- 保持当前界面密度与节奏

规则：

- 使用当前基础字号
- 不调整卡片与控件尺寸

#### 模式 B：大字体

用途：

- 面向大多数用户的轻量放大需求

规则：

- 页面基础字号整体放大一档
- 标题和正文同比放大
- 控件高度略增
- 胶囊与按钮略增高

#### 模式 C：易读模式

用途：

- 面向中老年用户或视力偏弱用户

规则：

- 字体明显放大
- 行高同步增大
- 按钮高度明显增大
- 胶囊高度增大
- 卡片内边距略增
- 列表项上下间距略增
- 关键按钮点击区更大

### 4.2 模式差异建议

建议不要只做 `font-size` 变化，而是至少联动以下维度：

- `--page-font-size-base`
- `--page-font-size-title`
- `--page-font-size-caption`
- `--page-line-height-body`
- `--page-control-height`
- `--page-chip-height`
- `--page-card-padding`

## 六、数据模型与偏好存储

### 6.1 用户偏好新增字段

建议在 `users` 集合里的用户偏好中新增：

```json
{
  "reminderSettings": {
    "followUpEnabled": true,
    "followUpAdvance": "same_day",
    "taskEnabled": true,
    "taskAdvance": "same_day"
  },
  "appearanceSettings": {
    "themeKey": "deep_business",
    "fontScaleMode": "default",
    "festivalThemeEnabled": false
  }
}
```

### 6.2 字段说明

- `themeKey`
  主题标识
- `fontScaleMode`
  字体 / 阅读模式标识
- `festivalThemeEnabled`
  是否允许在特殊时期自动切换节日主题

### 6.3 默认值建议

```json
{
  "themeKey": "deep_business",
  "fontScaleMode": "default",
  "festivalThemeEnabled": false
}
```

原因：

- `深海商务` 最接近当前默认视觉
- 初始用户迁移成本最低

## 九、“我的 > 视觉主题”页面规划

### 9.1 页面结构建议

建议新增或扩展现有“视觉主题”设置区为两部分：

展示方式：

- 三选一单选项

展示方式：

- 一个开关
- 文案尽量简短，不做复杂说明

- 切换主题后立即局部预览
- 点击保存后全局生效
- 主题与字体模式相互独立
- 任何组合都必须可预览

## 十一、验收标准

### 11.1 视觉主题验收

1. 至少 4 套主题可稳定切换
2. 同一主题下核心页面视觉一致
3. 不出现某一页漏换色、状态色失真或按钮不可读

### 11.2 阅读模式验收

1. `默认 / 大字体 / 易读模式` 能稳定切换
2. 大字体不会导致关键按钮被截断
3. 易读模式下时间选择器、半窗、表单输入可正常使用

### 11.3 真机体验验收

至少验证：

- iPhone 小屏
- 主流安卓大屏
- Hero 页面
- 表单页
- 半窗弹层
- 时间线页
- 分享卡页

---

## 十二、当前建议结论

当前建议不是立刻写 5 套主题页面，而是按以下顺序推进：

1. 先补 `appearanceSettings` 数据结构
2. 先做字体与阅读模式
3. 再抽全局主题 Token
4. 再做 4 套常驻主题
5. 最后再做节日主题位

这样能保证每一步都有明确产出，不会把“小程序换肤”做成难维护的样式堆积工程。
