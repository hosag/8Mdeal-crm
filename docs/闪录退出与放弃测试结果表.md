# 闪录退出与放弃测试结果表

这份表是给真机测试直接填写用的。

使用方式：

1. 先看“代码预判”
2. 真机执行后，在“真机结果”里补 `通过 / 失败 / 待测`
3. 如果失败，在“问题记录”里写一句最短描述

判定说明：

- `代码已覆盖`：从当前代码路径看，这条逻辑已经落地
- `需真机验证`：代码有实现，但最终体验必须靠真机确认
- `高风险关注`：这条最容易在真机暴露边界问题

### D. 选择“保存草稿，稍后继续”后恢复
