# 闪录隐式项目记忆方案

## 目标

闪录中的“项目别名学习”不再作为独立功能暴露给用户，也不再要求用户进入单独页面整理。

产品目标改为：

1. 用户打开闪录时，只关心录入、识别、确认和保存。
2. 系统在后台静默学习用户已经确认过的口语叫法、识别偏差和高频缩写。
3. 下次同一账号再次说出相似内容时，项目召回和排序更稳定，尽量减少手动纠正次数。

## 产品原则

1. 不建设单独“别名管理”主流程，不增加用户认知负担。
2. 不盲目学习所有识别结果，避免学偏。
3. 只在足够可信的动作后写入记忆：
   - 用户手动纠正并确认项目
   - AI 高置信自动匹配，且可提取出明确短 token
4. 记忆默认按账号隔离，避免不同销售之间互相污染。
5. 记忆服务于“项目匹配”，不是长期配置中心。

## 当前实现

补充说明：

1. 为了兼容历史数据和已有云函数，当前数据库字段与部分代码字段仍保留 `voiceAliases` / `voiceAliasesText` 命名。
2. 这些字段现在只作为历史兼容层使用，不再代表独立的用户可见“别名管理功能”。
3. 产品语义已经切换为：
   - 项目线索：项目自带或历史沉淀下来的可识别短叫法
   - 项目记忆：系统在闪录过程中静默学习到的高价值口语线索
4. 首页本地命中记录内部枚举已迁移到 `project_signal` / `project_memory`，但仍兼容历史 `configured_alias` / `memory_alias` 记录。

### 1. 前台行为

首页闪录已经改为：

1. 不再展示“已学习 / 未学习 / 手动确认后会更容易命中”之类提示。
2. 不再暴露“别名整理”入口作为正常用户路径。
3. 用户只看到：
   - 已自动匹配，可更换
   - 未匹配到项目，请从项目库选择
   - 已找到候选项目，请确认

### 2. 记忆写入时机

当前有两类写入：

1. `manual_confirm`
   - 用户手动从项目库选择正确项目
   - 从本次语音原文 / 跟进内容中提取可学习短 token
   - 只写入通过校验的 token

2. `ai_high_confidence`
   - AI 返回高置信且直接命中项目
   - 仅提取少量明确 token
   - 强度低于手动确认，避免误学

### 3. 记忆读取时机

当首页加载闪录项目列表时：

1. 先加载项目列表
2. 再按当前账号 + 当前项目列表读取 `projectAliasMemories`
3. 合并到本地候选召回缓存
4. 后续本地筛选和 AI 候选排序自动享受这层记忆增益

云端 `resolveQuickEntryProject` 当前也会读取同一份 `projectAliasMemories`：

1. 先把项目记忆转成候选增强信号
2. 直接参与候选前置排序
3. 同时作为 `项目记忆` 上下文提供给模型复判
4. 再结合项目名 / 客户名 / 联系人 / 项目线索做一轮云端信号评分
5. 当第一名综合得分明显领先时，允许走保守的确定性兜底，减少本可自动命中的场景落回手动确认

### 4. 数据集合

新增集合：`projectAliasMemories`

建议字段：

```json
{
  "accountId": "acc_xxx",
  "projectId": "project_xxx",
  "aliasText": "华东制造",
  "normalizedAliasText": "华东制造",
  "sourceType": "manual_confirm",
  "hitCount": 3,
  "strength": 6,
  "enabled": true,
  "createdAt": "2026-05-06T10:00:00.000Z",
  "updatedAt": "2026-05-06T10:00:00.000Z",
  "lastUsedAt": "2026-05-06T10:00:00.000Z"
}
```

字段说明：

1. `accountId`
   - 账号隔离
2. `projectId`
   - 只对单个项目生效
3. `aliasText`
   - 原始记忆文本
4. `normalizedAliasText`
   - 用于去重
5. `sourceType`
   - `manual_confirm` / `ai_high_confidence` / `speech_variant`
6. `hitCount`
   - 被学习或再次确认次数
7. `strength`
   - 简化权重，用于排序
8. `enabled`
   - 预留熔断能力

## 云函数

### `getQuickEntryProjectMemory`

用途：

1. 按当前账号读取项目记忆
2. 返回 `memoriesByProjectId`
3. 前端合并进闪录本地召回缓存

### `rememberQuickEntryProjectMemory`

用途：

1. 写入或更新项目隐式记忆
2. 对同账号 + 同项目 + 同标准化文本做去重
3. 累加 `hitCount` 和 `strength`

## 风险与控制

### 1. 学偏风险

风险：

1. 语音识别偶发错误可能被错误写入
2. 不同项目短名过于相近时可能污染记忆

控制：

1. 手动纠正优先，权重更高
2. 高置信自动匹配只学少量短 token
3. token 必须通过最小长度和有效性校验
4. 后续可增加停用或衰减机制

### 2. 项目阶段性强

风险：

1. 某些项目只使用很短周期
2. 长期保留大量记忆会逐渐失效

控制：

1. 当前按项目隔离
2. 后续可根据 `lastUsedAt` 做清理或衰减

### 3. 多设备一致性

风险：

1. 只依赖本地缓存会导致设备之间不一致

控制：

1. 云端集合为主
2. 本地缓存仅作为闪录即时兜底
