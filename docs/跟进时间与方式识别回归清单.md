# 跟进时间与方式识别回归清单

适用范围：

- 首页闪录 `follow_up` 流
- 新增跟进页 `pages/follow-up/follow-up`
- 本地规则工具 `utils/follow-up-meta.js`
- 跟进摘要云函数 `cloudfunctions/summarizeFollowUp`

本清单用于回归这次“规则优先 + AI 兜底 + 手动锁定”的时间与方式识别逻辑，避免后续调整闪录 UI、AI prompt 或本地规则时把正确性回退掉。

## 本地脚本回归

执行命令：

```bash
node scripts/check_follow_up_meta.js
```

通过标准：

- 输出 `follow_up meta regression checks passed`
- 不出现异常堆栈

脚本基准时间固定为 `2026-05-23 16:20`，所以以下相对时间样例的预期值也以这个时间为准。

## 规则基准

方式识别：

- 明确提到 `电话 / 微信 / 邮件 / 面谈` 时，优先直接命中
- 面谈类描述包括：`去公司`、`来公司`、`见面`、`会面`、`一起吃饭`、`打球`
- 未明确提到方式时，默认 `其他`
- 如果同一句里只有未来计划方式，没有已发生方式，不应误命中
- 如果一句里同时包含“未来计划方式 + 已发生方式”，应只保留已发生方式
- 如果一句里同时出现多个已发生方式，宁可回落 `其他`，不要乱猜

时间识别：

- 未明确提到发生时间时，默认当前时间
- `昨晚` 等模糊时段应落成稳定时刻：
  - 凌晨 `02:00`
  - 早上 `09:30`
  - 上午 `10:00`
  - 中午 `12:00`
  - 下午 `15:00`
  - 傍晚 `18:30`
  - 晚上 `20:00`
- 只应识别“本次跟进已发生时间”，不能把未来计划时间当成发生时间
- 同日模糊时间不应反压 AI 的默认当前时间
- 历史日期或精确时间可以反压 AI 的默认当前时间

## 首页闪录手工回归

入口：

- 首页
- 打开快速录入
- 选择 `记跟进`

检查点：

1. 输入阶段，文本框保持稳定，命中项目只给轻提示，不跳大卡片。
2. 点击 `AI整理` 后，方式、发生日期、发生时间应按上表回填。
3. 若用户手动改过方式、日期、时间，再次触发 AI，不应覆盖手改值。
4. `review` 阶段不展示候选项目搜索或切换入口，只保留已锁定项目信息。
5. `原始记录` 只读展示，不出现“展开修改”“切回语音录入”“重新AI整理”。
6. 保存成功后只弹轻提示并自动关闭闪录面板，不出现大成功面板。

建议最少回归句单：

- `昨晚电话交流了半小时，客户说先内部评估`
- `昨天微信沟通，对方要我明天发报价`
- `5月3日去公司面谈，现场看了机房`
- `今天下午安排电话沟通`
- `今晚 8:30 给客户打电话`
- `明天来公司见面，今天先电话沟通了需求`

## 新增跟进页手工回归

入口：

- 任意项目详情页进入新增跟进
- 或直接打开 `pages/follow-up/follow-up`

检查点：

1. 点击 `AI整理` 后，方式、发生日期、发生时间应按上表回填。
2. 若用户先手动修改方式、日期或时间，再次 `AI整理` 不应覆盖。
3. AI 失败时，如果本地规则有高置信命中，应保留本地规则结果；否则保持当前值。
4. `恢复上一版` 现在只存在于 `AI 摘要`，不应改写用户已手改的方式/时间；`AI 下一步` 不再提供恢复上一版入口。
5. `AI 下一步` 面板当前只展示 `最优先动作` 与 `采用动作`，不应再出现建议对象、建议方式、建议时间、建议理由、建议动作草稿等解释层信息。
6. 点击 `采用动作` 后，应回填下次跟进日期/时间与任务草稿；保存时最终写入的 `method` 与 `followUpTime` 仍应与页面当前值一致。

## 变更后必跑

只要改到以下任一文件，都应至少跑一次脚本回归，并抽 2 到 3 条做手工验证：

- `utils/follow-up-meta.js`
- `pages/index/index.js`
- `pages/follow-up/follow-up.js`
- `cloudfunctions/summarizeFollowUp/index.js`

建议命令：

```bash
node scripts/check_follow_up_meta.js
node --check utils/follow-up-meta.js
node --check pages/index/index.js
node --check pages/follow-up/follow-up.js
node --check cloudfunctions/summarizeFollowUp/index.js
```
