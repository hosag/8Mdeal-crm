## 一、实施目标

本轮实施最终要达到：

1. 首页、项目页、外发页、我的页都能原地打开同一套闪录半窗
2. 关闭、保存草稿、放弃本次闪录时，不再依赖跳首页承载
3. 闪录核心状态逐步从首页脚本中抽出，迁移到共享控制层
4. 保持现有语音识别、AI 理解、项目匹配、草稿恢复链路可用

## 七、建议开发顺序

为了降低风险，建议按下面顺序执行：

1. `B1` 创建组件空壳
2. `B2` / `B3` 首页组件化与 adapter
3. `B4` / `B5` / `B6` 三个页面接入
4. `B7` 关闭协议统一
5. 阶段 B 回归
6. `C1` 到 `C6` 控制层迁移
7. 阶段 C 回归
8. `D1` 到 `D4` 清理与全量验证

不建议的顺序：

- 先删首页旧逻辑，再补组件
- 一口气把语音、AI、草稿、UI 同时全部改掉
- 页面各自先接一套临时闪录逻辑再合并
