## 一、代码侧检查

- [ ] [cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js) 中的 `envId` 已替换为真实云环境 ID
- [ ] [cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js) 中的 `useMock` 已改为 `false`

## 三、微信开发者工具操作顺序

按这个顺序点，不要跳步：

1. 打开微信开发者工具，选择“导入项目”
2. 项目目录选择 [成交CRM-CodeX版](/Users/shaominhe/成交CRM-CodeX版)
3. 确认导入后使用的 `AppID` 与 [project.config.json](/Users/shaominhe/成交CRM-CodeX版/project.config.json) 一致
4. 打开项目后，先不要急着预览页面，先进入“云开发”面板
5. 如果还没有环境，先创建云开发环境；如果已有环境，记下环境 ID
6. 回到代码，修改 [cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js)
7. 保存后回到开发者工具，让项目重新编译
8. 进入 `cloudfunctions` 目录，对 `getDemoData` 执行上传并部署
9. 再依次上传并部署 `login`、`createNotifyTask`、`sendNotify`
10. 进入“云数据库”，逐个创建需要的集合
11. 回到模拟器，打开首页或我的项目页
12. 看页面顶部右侧的数据源标识

## 四、接通成功的判断标准

满足下面 3 条，才算真正进入“云环境就绪”状态：

- [ ] 页面顶部状态不再显示 `Mock Demo`
- [ ] 页面顶部状态显示 `CloudBase 已连接`
- [ ] 控制台没有出现 `CloudBase unavailable` 之类的错误

## 五、常见失败点

### 看到 `Mock Demo · 待填写 Env ID`

说明：
[cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js) 里的 `envId` 还是占位值。

处理：
把 `YOUR_CLOUD_ENV_ID` 换成真实环境 ID。

### 看到 `Mock Demo · 已开启模拟数据`

说明：
`useMock` 还没关。

处理：
把 [cloud.js](/Users/shaominhe/成交CRM-CodeX版/config/cloud.js) 里的 `useMock` 改成 `false`。

### 页面仍能打开，但一直不是 CloudBase

说明：
项目还在走本地 mock 回退逻辑，通常是云函数未部署、环境未选对，或者 `AppID` / `envId` 不匹配。

处理顺序：
1. 先检查 `envId`
2. 再检查 `useMock`
3. 再检查 `appid`
4. 最后检查云函数是否真的部署成功
