# 个人CRM小程序 - 项目规划文档（PRD）

> 版本：V1.1（根据当前实现更新）
> 适用行业：IT / SaaS / 咨询服务
> 技术方案：微信小程序云开发（CloudBase）
> 更新日期：2026-04-13

---

## 一、项目背景与目标

### 1.1 背景
业务人员在日常项目跟进中，存在以下痛点：
- 公司CRM系统偏重「汇报」，不方便个人灵活管理项目细节；
- 在对外分享项目信息或正式外发项目时，缺乏一个既快捷又能控制信息暴露程度的工具；
- 成交后的提成预期等个人收益难以追踪。

### 1.2 产品定位
一款**面向业务人员个人的轻量级CRM小程序**，核心定位：
- **个人作战工具**：独立于公司系统，数据归个人所有；
- **项目型销售导向**：支持一个项目下管理多个联系人（决策人、技术对接人等）；
- **灵活分享**：通过「分享卡片」实现“分享信息”和“项目外发”两种固定场景，按标签控制可见字段；
- **收益可视化**：成交后记录合同金额与预期提成，随时查看收益统计。

### 1.3 成功指标（MVP阶段）
- 用户可以完成「项目创建 → 录入跟进 → 生成分享卡片 → 成交记录」的完整闭环；
- 分享卡片接收者能在小程序内查看授权信息；
- 用户能在首页时间线中快速感知当日/近期待办；
- 项目外发与分享信息两种场景均可通过小程序卡片安全转发。

## 三、功能模块规划

### 3.1 项目管理中心

#### 3.1.1 项目列表
- **视图切换**：卡片视图 / 列表视图（默认卡片视图，展示项目阶段+最近跟进时间+预计金额）；
- **筛选与搜索**：按项目阶段（线索/洽谈/方案/商务/成交/流失）、按客户名称/项目名称搜索；
- **快捷操作**：点击进入项目详情，支持录入跟进、编辑项目。

#### 3.1.2 项目详情
- **顶部信息卡**：项目名称、客户名称、当前阶段、预计金额、成交金额、预期提成；
- **联系人模块**：展示该项目下所有联系人（姓名、角色、电话、微信、所属公司等），支持快捷拨打/复制；联系人信息以**底部弹窗（bottom-sheet）**形式展示；
- **跟进时间线**：按日期分组展示所有跟进记录（今天/昨天/具体日期），支持图文混排，显示明确时间；
- **底部操作栏**：录入跟进、编辑项目、分享配置、标记成交（权限控制下展示）。

#### 3.1.3 新建/编辑项目
- **必填项**：项目名称、客户名称、当前阶段；
- **选填项**：预计金额、项目描述、标签；
- **联系人**：支持在项目创建时直接添加多个联系人，也支持后续补充。

---

### 3.2 联系人管理

采用**项目内嵌**设计：
- 每个项目下独立维护联系人列表；
- 联系人字段：姓名、角色（决策人/技术对接人/其他）、电话、微信、邮箱、所属公司、备注；
- 在项目详情页点击联系人区域，弹出底部面板展示完整联系方式。

---

### 3.3 跟进记录

#### 3.3.1 跟进字段
- 跟进时间（默认当前时间，可修改）；
- 跟进方式（电话/微信/邮件/面谈/其他）；
- 跟进内容（文本）；
- 附件（图片，最多9张）；
- 阶段变更（可选，若此次跟进导致项目阶段变化，可同步更新）。

#### 3.3.2 跟进入口
- 项目详情页底部「录入跟进」；
- 首页时间线中快速查看近期跟进项目。

#### 3.3.3 提醒机制
- 用户在录入跟进时可设置「下次跟进时间」；
- 系统通过微信一次性订阅消息，在设定时间前提醒用户；
- 首页展示逾期未跟进项目（标红提示）。

---

### 3.4 分享卡片与权限控制（核心功能）

#### 3.4.1 分享场景
小程序分享分为两种固定模式，不再支持自定义海报：

| 场景 | 名称 | 分享对象 | 展示内容规则 | 管理权归属 |
|------|------|---------|-------------|-----------|
| 查看型分享 | 分享信息 | 需要了解项目情况的人 | 展示项目信息，**隐藏客户电话/微信等联系方式** | 仅分享者拥有管理权 |
| 正式外发 | 项目外发 | 需要接手项目的人 | 展示项目信息+**全部联系方式** | 接收者在「我的项目」中获得**完整管理权** |

#### 3.4.2 分享对象标签
系统提供默认标签，用户可在「隐私标签」页面进行增删改：
- **基础浏览**（默认）：隐藏客户电话、微信等联系方式；
- **完整外发**（默认）：展示全部信息，包括联系方式；
- **全量查看**（默认）：展示全部信息，包括联系方式。

#### 3.4.3 分享卡片形态
- **仅支持小程序卡片转发**：用户点击「转发给好友」后触发系统分享菜单，接收者打开小程序卡片进入「分享卡片页」；
- 分享卡片页仅展示被授权标签过滤后的字段；
- 卡片页底部显示「分享来源：XXX」，并提示「如有兴趣请联系我」。

#### 3.4.4 权限模型说明
- **我的项目（Tab2）**：包含用户自己创建的项目 + 其他用户通过「项目外发」共享给该用户的项目。外发进来的项目享有**完整的编辑、跟进、成交标记权限**，但会显示紫色「外发」来源徽章；
- **外发项目（Tab3）**：用户通过「项目外发」分享出去的项目列表，采用**只读视图**，方便追踪自己共享出去的商机；
- **分享信息**：不会转移管理权，接收者仅能在卡片页查看被过滤后的信息。

---

### 3.5 首页总览（时间线 + 待办提醒）

#### 3.5.1 时间线视图（默认首页）
- **今日待办卡片**：置顶展示今日需要跟进的项目（基于用户设置的「下次跟进时间」）；
- **动态时间线**：按日期分组展示「跟进记录录入」、「项目阶段变更」、「新项目的创建」、「成交记录」等关键事件；
- **快捷操作**：每条待办可直接点击进入项目详情或快捷录入跟进。

#### 3.5.2 数据汇总区（首页顶部）
- 本月新增项目数；
- 本月成交金额 / 预期提成总额；
- 待跟进项目数（含逾期）。

---

### 3.6 成交与提成管理

#### 3.6.1 成交标记
- 用户在项目详情页点击「标记成交」，进入成交记录页；
- 填写：实际成交金额、合同签订日期、回款状态（未回款/部分回款/全部回款）、预期提成金额（手动填写）。

#### 3.6.2 提成总览
- 「我的」页面中提供「收益统计」入口；
- 按月份/季度统计：成交金额、预期提成总额、已回款金额；
- 统计维度：按项目列表展示每笔成交的提成预期。

### 4.2 安全策略
- 所有数据库操作通过云函数中转，避免前端直接操作敏感数据；
- 分享卡片的数据通过云函数按「shareTagId」过滤后再返回，确保未授权字段不可见；
- 用户只能查询 `_openid` 等于自己的项目、跟进记录和成交记录（分享页除外）；
- 「项目外发」共享进来的项目，通过 `shareRecords` 关联查询获得管理权限。

| **录入跟进页** (`pages/follow-up/follow-up`) | 录入跟进内容、上传图片、设置下次跟进时间 |

| **Tab3: 外发项目** (`pages/shared-out/shared-out`) | 用户外发出去的项目列表（只读视图） |
| **Tab4: 我的** (`pages/mine/mine`) | 个人资料、收益统计、隐私标签设置 |
| **成交记录页** (`pages/mark-deal/mark-deal`) | 标记成交、填写金额与提成 |

| **隐私标签页** (`pages/privacy-tags/privacy-tags`) | 管理分享标签（增删改） |
| **编辑标签页** (`pages/edit-tag/edit-tag`) | 新建或编辑单个标签的可见字段 |

### 5.2 核心流程图（文字版）

#### 流程A：创建项目并录入跟进
```
首页/我的项目 → 新建项目 → 填写信息/添加联系人 → 保存 → 项目详情 → 录入跟进 → 保存跟进（可选设置下次跟进时间） → 返回详情/首页
```

#### 流程B：分享项目卡片
```
项目详情 → 分享配置 → 选择模式（分享信息 / 项目外发） → 选择标签 → 预览卡片 → 转发给微信好友 → 接收者打开小程序卡片页 → 云函数按标签过滤返回数据
```

#### 流程C：成交记录与提成
```
项目详情 → 标记成交 → 填写成交金额/预期提成 → 保存 → 收益统计更新
```

## 六、技术方案

### 6.1 技术栈
- **前端**：微信小程序原生框架（WXML + WXSS + JS）；
- **后端**：微信云开发（CloudBase），使用云数据库 + 云函数 + 云存储；
- **UI组件**：Vant Weapp（`@vant/weapp`），组件丰富，适合快速开发；
- **图表**：收益统计页面使用简单数据展示（后续可扩展 ECharts/F2）。

### 6.2 云开发架构
```
小程序端
  ├─ 首页（时间线/待办/汇总）
  ├─ 我的项目（列表/详情/跟进/成交/编辑）
  ├─ 外发项目（只读列表）
  ├─ 分享页（接收者卡片）
  └─ 我的页（设置/统计/标签管理）

云函数
  ├─ login: 用户登录与自动注册
  ├─ createNotifyTask: 创建订阅消息提醒任务
  └─ sendNotify: 定时触发，轮询并发送订阅消息

云数据库
  ├─ users
  ├─ projects
  ├─ followUps
  ├─ shareRecords
  ├─ deals
  └─ notifications

云存储
  └─ 跟进图片附件、用户头像
```

### 6.3 订阅消息模板
- **提醒类型**：使用「一次性订阅消息」；
- **触发时机**：用户在录入跟进时设置「下次跟进时间」，`createNotifyTask` 云函数在 `notifications` 集合中创建任务；
- **发送机制**：`sendNotify` 云函数通过定时触发器每小时轮询，到达提醒时间后发送订阅消息；
- **模板建议**：
  - 类目：商业服务 / 企业管理
  - 模板关键词：客户名称、跟进事项、提醒时间

## 八、风险与应对

| 风险点 | 影响 | 应对策略 |
|--------|------|---------|
| 微信小程序订阅消息触达率有限 | 待办提醒可能失效 | 首页强依赖时间线展示，提醒作为辅助；同时支持用户手动设置日历提醒 |
| 分享卡片的数据安全（前端可能被破解） | 敏感信息泄露 | 所有分享数据必须通过云函数按标签过滤后返回，前端不持有完整项目数据 |
| 云开发免费额度不足 | 图片存储/调用量超限 | 前期控制图片压缩质量；关注用量，必要时升级付费套餐 |
| 用户习惯养成难 | 个人CRM使用频率低 | 降低录入成本（快捷跟进），强化首页待办的压迫感 |
